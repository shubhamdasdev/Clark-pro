# Evidence Brief Review

Use Clark's already trusted, zero-egress idea inspection capability to review whether an idea brief states its thesis clearly and separates structure from observed proof.

## Procedure

1. Read only the exact idea revision selected by the current run.
2. Invoke `clark.idea.inspect.mcp` only through the Harness capability boundary.
3. Report explicit thesis facets, missing facets, and unobserved evidence gates.
4. Treat structural completeness as readiness to collect evidence, never as market validation.
5. Return the report to the current run for creator review.

## Authority

- No network, credential, filesystem, model, memory mutation, external write, or publication authority.
- No capability may be inferred from these instructions.
- The current run's policy and approval remain authoritative.
- Skill promotion establishes revision trust only; each invocation still requires an effective permission receipt.

## Failure behavior

Stop if the selected revision, capability revision, workspace scope, or expected structured result cannot be verified exactly. Do not substitute a newer revision or silently degrade to another tool.
